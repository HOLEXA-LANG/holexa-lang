# Contributing to HOLEXA

Thank you for your interest in contributing to HOLEXA!

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR-USERNAME/holexa-lang.git
   cd holexa-lang/compiler
   make
   ```

3. Make your changes
4. Run tests:
   ```bash
   make test
   ```
5. Submit a pull request

## Areas to Contribute

- **Compiler** — lexer, parser, semantic analysis, codegen
- **Standard Library** — new stdlib modules
- **Documentation** — guides, examples, API docs
- **Editor Support** — VS Code, Neovim, etc.
- **Tests** — more test cases
- **Examples** — real-world HOLEXA programs

## Code Style

- C99 for the compiler
- Clear variable names
- Comments for complex logic
- Tests for all new features

## Reporting Bugs

Use [GitHub Issues](https://github.com/HOLEXA-LANG/holexa-lang/issues).
Include: HOLEXA version, OS, code that caused the issue, expected vs actual output.
